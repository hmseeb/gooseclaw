3. Rewrite `docker/tests/conftest.py` to include BOTH the existing source-inspection fixtures (gateway_source, entrypoint_source, secret_sh_source, tmp_data_dir) AND new HTTP-level fixtures:

**Keep existing fixtures** (Phase 18 tests depend on them):
- `tmp_data_dir` - temp dir with SHA-256 hashed password setup.json
- `gateway_source` - gateway.py source as string
- `entrypoint_source` - entrypoint.sh source as string
- `secret_sh_source` - secret.sh source as string

**Add new fixtures:**

a) `gateway_module` (scope="session") - Set DATA_DIR, PORT env vars before importing gateway. Return the module. Session-scoped to avoid re-import.

b) `live_gateway` (scope="module") - Using `gateway_module`:
   - Create tmp dirs: data/, data/config/, data/secrets/ (0o700), data/sessions/, data/channels/
   - Create empty data/secrets/vault.yaml
   - Patch module globals: DATA_DIR, CONFIG_DIR
   - Instantiate ThreadingHTTPServer(("127.0.0.1", 0), gw.GatewayHandler)
   - Start in daemon thread
   - Poll /api/health until ready (max 50 attempts, 50ms sleep)
   - yield base_url string
   - Teardown: server.shutdown(), restore original globals

c) `auth_session` (scope="function") - Takes live_gateway. Creates a PBKDF2-hashed password in the gateway's config dir setup.json, then POSTs to /api/auth/login with password "testpassword". Returns dict with Cookie header for authenticated requests.

d) `reset_rate_limiters` (autouse=True, scope="function") - Clears auth_limiter.requests, api_limiter.requests, and notify_limiter.requests dicts on the gateway_module before each test.

**Critical details:**
- Import gateway INSIDE the gateway_module fixture, not at module level (module-level side effects)
- Use `_requests` dict (the internal attribute) to clear rate limiters, since that's what RateLimiter stores timestamps in
- The auth_session fixture must compute a PBKDF2 hash using the gateway module's `hash_token` function to write to setup.json
- Keep `tmp_data_dir` fixture as-is for backward compat with Phase 18 source-inspection tests

4. Install dev dependencies: `cd docker && pip install -r requirements-dev.txt`
