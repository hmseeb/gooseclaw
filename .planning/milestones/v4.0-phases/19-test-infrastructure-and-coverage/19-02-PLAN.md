**Keep all existing source-inspection test classes** (TestPBKDF2HashFormat, TestLegacySHA256Migration). Add new HTTP-level test classes below them.

Add these test classes using live_gateway and auth_session fixtures:

**class TestAuthLogin:**
- `test_login_correct_password(live_gateway)` - Set up a password in the gateway's config (write setup.json with PBKDF2 hash via gateway module's hash_token), POST /api/auth/login with {"password": "testpassword"}, assert 200 and Set-Cookie header present
- `test_login_wrong_password(live_gateway)` - POST /api/auth/login with {"password": "wrongpassword"}, assert 401
- `test_login_missing_password(live_gateway)` - POST /api/auth/login with {}, assert 400 or 401
- `test_login_empty_body(live_gateway)` - POST /api/auth/login with no body, assert 400 or 401

**class TestAuthSession:**
- `test_authenticated_endpoint_with_cookie(live_gateway, auth_session)` - GET /api/setup/config with auth cookie, assert 200 (not 401)
- `test_authenticated_endpoint_without_cookie(live_gateway)` - GET /api/setup/config without cookie (after setup_complete is True), assert 401
- `test_session_cookie_httponly(live_gateway)` - Login and check Set-Cookie header contains HttpOnly flag

**class TestAuthRateLimiting:**
- `test_auth_rate_limit_triggers(live_gateway, gateway_module)` - Clear rate limiter, send auth_limiter.max_requests + 1 login attempts (wrong password), assert the last one returns 429. Read max_requests from the gateway module's auth_limiter instance.

**class TestAuthRecovery:**
- `test_recovery_with_valid_secret(live_gateway, gateway_module)` - Write a recovery secret file to the gateway's data dir (DATA_DIR/recovery_secret), POST /api/auth/recover with {"recovery_secret": "...", "new_password": "newpass"}, assert 200. Then verify login works with new password.
- `test_recovery_with_invalid_secret(live_gateway)` - POST /api/auth/recover with wrong secret, assert 401 or 403

**Important:** Each test that needs a password set up must write setup.json to the gateway module's current DATA_DIR/config/ path. Use the gateway module's `hash_token()` function to compute PBKDF2 hashes. Access the gateway module via `gateway_module` fixture or import it from the `gateway_module` fixture indirectly.

**Important:** The reset_rate_limiters fixture (autouse) ensures each test starts with clean rate limiter state.

**Keep all existing source-inspection test classes** (TestSecretShInjection, TestEntrypointInjection, TestRunScriptInjection, TestRecoverySecretLeak, TestBodySizeLimit, TestSecurityHeaders). Add new HTTP-level test classes below them.

Add these test classes:

**class TestHTTPSecurityHeaders:**
Define a dict of expected headers matching gateway's SECURITY_HEADERS:
```python
EXPECTED = {
    "X-Content-Type-Options": "nosniff",
    "X-Frame-Options": "DENY",
    "X-XSS-Protection": "1; mode=block",
    "Referrer-Policy": "strict-origin-when-cross-origin",
    "Permissions-Policy": "camera=(), microphone=(), geolocation=()",
    "Cross-Origin-Opener-Policy": "same-origin",
}
```

- `test_security_headers_on_health(live_gateway)` - GET /api/health, assert all EXPECTED headers present
- `test_security_headers_on_setup_page(live_gateway)` - GET /setup, assert all EXPECTED headers present
- `test_security_headers_on_404(live_gateway)` - GET /nonexistent, assert all EXPECTED headers present (even error responses)
- `test_security_headers_on_auth_endpoint(live_gateway)` - POST /api/auth/login, assert all EXPECTED headers present
- `test_content_type_json_on_api(live_gateway)` - GET /api/health, assert Content-Type contains application/json

Create a helper function `assert_security_headers(response)` that checks all headers and gives clear error messages on failure. Use it in each test.

**class TestHTTPCORS:**
- `test_cors_preflight_allowed_origin(live_gateway)` - Send OPTIONS request with Origin header, assert Access-Control-Allow-Origin is present
- `test_cors_preflight_methods(live_gateway)` - OPTIONS request includes Access-Control-Allow-Methods
- `test_cors_actual_request_has_origin(live_gateway)` - GET /api/health with Origin header, check Access-Control-Allow-Origin in response

**class TestHTTPBodyLimit:**
- `test_oversized_body_rejected(live_gateway)` - POST /api/auth/login with body larger than 1MB (send 1.1MB of "x" chars), assert 413 status
- `test_normal_body_accepted(live_gateway)` - POST /api/auth/login with normal-sized body, assert NOT 413 (400 or 401 are fine, just not 413)
