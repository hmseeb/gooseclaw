  <name>Task 1: Create generator module with string.Template engine</name>
  <files>docker/extensions/__init__.py, docker/extensions/generator.py, docker/extensions/templates/base_helpers.py.tmpl</files>
  <action>
Create the docker/extensions/ package:

1. **docker/extensions/__init__.py** -- empty package init.

2. **docker/extensions/templates/base_helpers.py.tmpl** -- a shared snippet that every generated server includes at the top. Contains:
   - Docstring with generation metadata: `"""Auto-generated MCP extension: ${extension_name}\nGenerated: ${generated_at}\nTemplate: ${template_name}\nVault keys: ${vault_keys_display}\n"""`
   - `import sys` and stdout-to-stderr redirect: `sys.stdout = sys.stderr` (TMPL-05 requirement -- prevents tool print() from corrupting MCP JSON-RPC on stdout)
   - `import subprocess` for vault access
   - `_vault_get(key)` function that calls `subprocess.run(["secret", "get", key], capture_output=True, text=True)`, checks returncode, raises RuntimeError on failure, returns `.stdout.strip()`
   - `from mcp.server.fastmcp import FastMCP` import
   - `mcp = FastMCP("${extension_name}")` initialization
   - A trailing `# --- template tools below ---` comment marker

3. **docker/extensions/generator.py** -- the generation engine:
   - `generate_extension(template_name: str, extension_name: str, vault_prefix: str, vault_keys: list[str], service_description: str = "") -> str`
     - Looks up template at `/app/docker/extensions/templates/{template_name}.py.tmpl`
     - Reads the template file content as a string
     - Reads `base_helpers.py.tmpl` content
     - Builds substitution dict: `extension_name`, `template_name`, `vault_prefix`, `vault_keys_display` (comma-joined vault_keys), `generated_at` (ISO timestamp), `service_description`, plus each vault key as a named variable (e.g., if vault_keys=["fastmail.imap_host", "fastmail.username"], the dict gets `vault_key_0="fastmail.imap_host"`, `vault_key_1="fastmail.username"`)
     - Uses `string.Template(base_helpers_content + "\n" + template_content).safe_substitute(subs)` to render
     - Creates output directory: `/data/extensions/{extension_name}/`
     - Writes rendered content to `/data/extensions/{extension_name}/server.py`
     - Makes server.py executable (chmod 755)
     - Returns the full path to the generated server.py
   - `list_templates() -> list[str]` -- lists available template names by scanning the templates/ directory for *.py.tmpl files (excluding base_helpers)
   - All logging to stderr via `logging.basicConfig(stream=sys.stderr)`
   - Use stdlib only (string, os, pathlib, datetime, stat, logging). No Jinja2. Research explicitly says string.Template is sufficient.

Important: The base_helpers template uses `${variable}` syntax which is Python string.Template's native syntax. Template files should use `${varname}` for substitution points. Literal dollar signs in generated code (like f-strings) must use `$$` to escape.
