Create docker/extensions/templates/email_imap.py.tmpl -- a complete FastMCP MCP server template for email via IMAP/SMTP.

This template file will be concatenated AFTER base_helpers.py.tmpl by the generator, so it should NOT include the imports/boilerplate already in base_helpers (no duplicate FastMCP init, no duplicate _vault_get, no duplicate sys.stdout redirect). It SHOULD include any email-specific imports.

Template content (using ${variable} for string.Template substitution, $$ for literal dollar signs):

1. **Email-specific imports** at the top:
   ```
   import imaplib
   import smtplib
   import email
   from email.mime.text import MIMEText
   from email.mime.multipart import MIMEMultipart
   from email.header import decode_header
   import json
   ```

2. **Helper: _get_imap_connection()** -- creates and returns an authenticated IMAP4_SSL connection using:
   - host from `_vault_get("${vault_prefix}.imap_host")`
   - username from `_vault_get("${vault_prefix}.username")`
   - password from `_vault_get("${vault_prefix}.app_password")`
   - Uses IMAP4_SSL (port 993 default)

3. **Helper: _get_smtp_connection()** -- creates and returns an authenticated SMTP_SSL connection using:
   - host from `_vault_get("${vault_prefix}.smtp_host")`
   - username from `_vault_get("${vault_prefix}.username")`
   - password from `_vault_get("${vault_prefix}.app_password")`
   - Uses SMTP_SSL (port 465 default)

4. **Helper: _decode_header_value(value)** -- safely decodes email header values handling charset.

5. **Tool: email_search(query: str, folder: str = "INBOX", limit: int = 10) -> str**
   - Docstring: "Search emails by subject, sender, or keyword."
   - Connect via _get_imap_connection()
   - Select folder
   - IMAP SEARCH with query (search by SUBJECT, FROM, or TEXT based on content)
   - Return JSON list of dicts with: uid, subject, from, date, snippet (first 200 chars of body)
   - Limit results to `limit`
   - Close connection in finally block

6. **Tool: email_read(uid: str, folder: str = "INBOX") -> str**
   - Docstring: "Read a specific email by UID. Returns full body text."
   - Connect, fetch by UID
   - Decode body (prefer text/plain, fallback text/html stripped)
   - Return JSON dict with: subject, from, to, date, body
   - Close connection in finally block

7. **Tool: email_send(to: str, subject: str, body: str) -> str**
   - Docstring: "Send an email."
   - Get sender from `_vault_get("${vault_prefix}.username")`
   - Create MIMEText message
   - Send via _get_smtp_connection()
   - Return confirmation string with recipient and subject
   - Close connection in finally block

8. **Server entry point** at bottom:
   ```
   if __name__ == "__main__":
       mcp.run()
   ```

Important notes:
- Use `$$` to escape literal dollar signs in f-strings (e.g., `f"Found $${len(results)} emails"` becomes `f"Found ${len(results)} emails"` after Template substitution)
- All error handling via try/except with descriptive error messages returned as strings (MCP tools should return error strings, not raise)
- All connections closed in finally blocks
- Only stdlib imports (imaplib, smtplib, email are all stdlib). No external email libraries.
