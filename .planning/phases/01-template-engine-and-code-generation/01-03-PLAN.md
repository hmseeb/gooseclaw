Create docker/extensions/templates/rest_api.py.tmpl -- a complete FastMCP MCP server template for generic authenticated REST API calls.

This template is concatenated AFTER base_helpers.py.tmpl by the generator. Do NOT duplicate base_helpers content (no duplicate _vault_get, no duplicate FastMCP init, no duplicate stdout redirect).

Template content (using ${variable} for string.Template substitution, $$ for literal dollar signs):

1. **REST-specific imports** at the top:
   ```
   import json
   import urllib.request
   import urllib.parse
   import urllib.error
   ```

2. **Helper: _build_headers(extra_headers: dict = None) -> dict** -- builds request headers:
   - Gets API key from `_vault_get("${vault_prefix}.api_key")`
   - Gets auth_type from template variable `${auth_type}` (default "bearer") -- values: "bearer", "api_key_header", "basic"
   - For "bearer": `{"Authorization": f"Bearer {api_key}"}`
   - For "api_key_header": `{"X-API-Key": api_key}` (also support custom header name via `${auth_header}` with default "X-API-Key")
   - Always adds `{"Content-Type": "application/json", "Accept": "application/json"}`
   - Merges extra_headers if provided
   - Returns combined headers dict

3. **Helper: _make_request(method: str, path: str, body: dict = None, params: dict = None, headers: dict = None) -> dict** -- core HTTP helper:
   - Gets base_url from `_vault_get("${vault_prefix}.base_url")`
   - Builds full URL: base_url.rstrip("/") + "/" + path.lstrip("/")
   - Appends query params if provided via urllib.parse.urlencode
   - Creates urllib.request.Request with method, headers from _build_headers(headers), and json body if provided
   - Executes request, reads response
   - Returns parsed JSON response or raw text
   - Catches urllib.error.HTTPError and returns error details as dict with status_code, reason, body

4. **Tool: api_get(path: str, params: str = "") -> str**
   - Docstring: "Make a GET request to ${service_description}. Params is a JSON object string of query parameters."
   - Parse params from JSON string if provided
   - Call _make_request("GET", path, params=parsed_params)
   - Return JSON string response

5. **Tool: api_post(path: str, body: str = "{}") -> str**
   - Docstring: "Make a POST request to ${service_description}. Body is a JSON object string."
   - Parse body from JSON string
   - Call _make_request("POST", path, body=parsed_body)
   - Return JSON string response

6. **Tool: api_put(path: str, body: str = "{}") -> str**
   - Docstring: "Make a PUT request to ${service_description}. Body is a JSON object string."
   - Same pattern as api_post but with PUT

7. **Tool: api_delete(path: str) -> str**
   - Docstring: "Make a DELETE request to ${service_description}."
   - Call _make_request("DELETE", path)
   - Return JSON string response

8. **Server entry point** at bottom:
   ```
   if __name__ == "__main__":
       mcp.run()
   ```

Important notes:
- Use `$$` to escape literal dollar signs in f-strings
- Only stdlib imports: urllib.request, urllib.parse, urllib.error, json. NO requests library (GEN-03 requirement).
- All error handling returns descriptive error strings, does not raise
- Template variables needed: ${vault_prefix}, ${auth_type} (default "bearer"), ${auth_header} (default "X-API-Key"), ${service_description}, ${extension_name} (from base_helpers)
- The generator.py generate_extension function may need minor updates to pass auth_type and auth_header as additional substitution variables with defaults. Add default values in generator.py: auth_type="bearer", auth_header="X-API-Key" if not provided in the substitution dict.
