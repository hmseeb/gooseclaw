<task type="auto">
  <name>Task 1: Create registry.py with CRUD operations and atomic file writes</name>
  <files>docker/extensions/registry.py</files>
  <action>
Create docker/extensions/registry.py implementing the extension registry.

Module-level constant:
- REGISTRY_PATH = "/data/extensions/registry.json" (monkeypatch-friendly for tests, same pattern as generator.py's OUTPUT_BASE_DIR)

Functions to implement:

1. `_load_registry()` - Load registry from disk. Returns `{"version": 1, "extensions": {}}` if file missing or corrupt. Use json.load().

2. `_save_registry(data)` - Atomic write with file locking:
   - os.makedirs for parent dir (exist_ok=True)
   - Write to tmp file (REGISTRY_PATH + ".tmp")
   - fcntl.flock(f, fcntl.LOCK_EX) on the tmp file for cross-process safety
   - json.dump with indent=2 for human readability
   - f.flush() + os.fsync(f.fileno()) for durability
   - os.replace(tmp_path, REGISTRY_PATH) for atomic swap

3. `register(name, template, vault_prefix, vault_keys, server_path, description="")` - Add/update extension entry:
   - Load registry, set extensions[name] to dict with fields: template, extension_name, vault_prefix, vault_keys (list), server_path, description, generated_at (UTC ISO), enabled (True)
   - Save registry
   - Return the entry dict

4. `unregister(name, delete_files=False)` - Remove extension:
   - Load registry, pop extensions[name]
   - If delete_files=True and entry existed: remove server_path file, rmdir parent if empty
   - Save registry
   - Return removed entry or None

5. `list_extensions()` - Return dict of all registered extensions (the "extensions" key from registry).

6. `get_config_entries()` - Return goosed config.yaml extension dicts for all ENABLED extensions:
   - For each entry where enabled=True, produce dict with exact goosed fields: enabled, type ("stdio"), name, description, cmd ("python3"), args ([server_path]), envs ({}), env_keys ([]), timeout (300), bundled (None), available_tools ([])
   - Skip disabled entries
   - Return dict keyed by extension name

Use only stdlib: json, os, fcntl, datetime. Follow the same logging pattern as generator.py (logging to stderr).
