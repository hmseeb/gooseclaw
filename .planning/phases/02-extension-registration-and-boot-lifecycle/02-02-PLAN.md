<task type="auto">
  <name>Task 2: Add registry boot loader to entrypoint.sh</name>
  <files>docker/entrypoint.sh</files>
  <action>
Insert a registry loading block into entrypoint.sh AFTER line 541 (`rm -f "$EXTENSIONS_STATE_FILE"`) and BEFORE the template extension sync comment at line 543 (`# --- sync new template extensions into existing configs`).

The block reads /data/extensions/registry.json and injects enabled extensions into config.yaml before goosed starts.

Insert this block:

```bash
# --- auto-generated extensions (from registry.json) ----------------------------------------
REGISTRY_FILE="/data/extensions/registry.json"
if [ -f "$REGISTRY_FILE" ]; then
    echo "[mcp] loading auto-generated extensions from registry..."
    python3 -c "
import yaml, json, os, sys, subprocess
try:
    with open('$REGISTRY_FILE') as f:
        registry = json.load(f)
    with open('$CONFIG_DIR/config.yaml') as f:
        config = yaml.safe_load(f) or {}
    exts = config.setdefault('extensions', {})
    added = []
    skipped = []
    for name, meta in registry.get('extensions', {}).items():
        if not meta.get('enabled', True):
            skipped.append(f'{name} (disabled)')
            continue
        sp = meta.get('server_path', '')
        if not os.path.isfile(sp):
            skipped.append(f'{name} (server.py missing)')
            continue
        exts[name] = {
            'enabled': True,
            'type': 'stdio',
            'name': name,
            'description': meta.get('description', f'Auto-generated {name} extension'),
            'cmd': 'python3',
            'args': [sp],
            'envs': {},
            'env_keys': [],
            'timeout': 300,
            'bundled': None,
            'available_tools': [],
        }
        added.append(name)
    if added:
        config['extensions'] = exts
        with open('$CONFIG_DIR/config.yaml', 'w') as f:
            yaml.dump(config, f, default_flow_style=False, sort_keys=False)
        print(f'[mcp] auto-generated extensions loaded: {\", \".join(added)}')
    if skipped:
        print(f'[mcp] auto-generated extensions skipped: {\", \".join(skipped)}', file=sys.stderr)
except Exception as e:
    print(f'[mcp] WARN: registry load failed: {e}', file=sys.stderr)
" 2>/dev/null || true
fi
```

Key points:
- Validates server.py exists on disk before injecting (handles stale registry entries)
- Skips disabled extensions
- Uses same config.yaml read-modify-write pattern as the template sync block below it
- Fails gracefully (try/except + `|| true`) so a corrupt registry never blocks boot
- Logs what was added and what was skipped
- Does NOT validate vault keys at boot (removed from research suggestion -- vault may not be ready at this point in boot, and the extension will fail gracefully at runtime if keys are missing)

IMPORTANT: Do not modify any existing lines. This is a pure insertion between the `rm -f "$EXTENSIONS_STATE_FILE"` line and the sync comment.
