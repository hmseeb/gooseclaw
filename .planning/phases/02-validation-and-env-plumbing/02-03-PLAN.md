Review and fix dispatch_validation() in gateway.py for all 23 providers. The frontend sends credentials in different shapes depending on the provider:

**Frontend sends these field names (from validateKey in setup.html):**
- Standard providers: `{provider_type, api_key}`
- claude-code: `{provider_type, claude_setup_token}`
- custom: `{provider_type, url, api_key}`
- ollama/local: `{provider_type, ollama_host}`
- azure-openai: `{provider_type, api_key, azure_endpoint}`
- github-copilot: `{provider_type}` (no credentials)

**Current issues to fix:**
1. The `credentials` parameter in dispatch_validation comes from `data.get("credentials", data)` -- when frontend sends flat payload (not nested under "credentials"), credentials == the whole data dict. This works but field name extraction needs to handle both flat and nested formats.

2. For azure-openai validation: frontend sends `azure_endpoint` but the handler looks for `AZURE_OPENAI_ENDPOINT` or `endpoint`. Add `azure_endpoint` as a fallback:
```python
endpoint = credentials.get("AZURE_OPENAI_ENDPOINT") or credentials.get("azure_endpoint") or credentials.get("endpoint", "")
```

3. For local providers: frontend sends `ollama_host` but handler looks for `OLLAMA_HOST` or `host` or `url`. Add `ollama_host` as a fallback:
```python
host = credentials.get("OLLAMA_HOST") or credentials.get("ollama_host") or credentials.get("host") or credentials.get("url")
```

4. For LiteLLM: frontend needs to send litellm_host. Add `litellm_host` as fallback:
```python
host = credentials.get("LITELLM_HOST") or credentials.get("litellm_host") or credentials.get("host", "")
```

5. Ensure CRED-05 compliance: Claude-code's response should include explicit instructions:
```python
if provider == "claude-code":

        "message": "Claude Code uses OAuth authentication. Run 'claude setup-token' in your local terminal to get the token. Validation must be done manually after saving.",
        "skip_validation": True,
    }
```

6. Ensure all validation error messages include the provider name and a link to the key URL where applicable. Current messages already do this for most providers -- verify nothing was missed.

7. Add validation for github-copilot GITHUB_TOKEN if provided (optional key):

if provider == "github-copilot":
    token = credentials.get("GITHUB_TOKEN") or credentials.get("api_key", "")
    if token:
        # token provided, try to validate via GitHub API
        headers = {"Authorization": f"Bearer {token}"}
        try:
            status, _ = http_get("https://api.github.com/copilot_internal/v2/token", headers=headers)

                return {"valid": True, "message": "GitHub Copilot token is valid."}

                return {"valid": False, "error": "Invalid GitHub token. Check your token or Copilot subscription."}
