  <name>Task 1: Create validator.py with syntax check, health check, and failure tracking</name>
  <files>docker/extensions/validator.py</files>
  <action>
Create docker/extensions/validator.py with these functions:

1. **validate_syntax(server_path: str) -> tuple[bool, str]**
   - Read the file at server_path
   - Run ast.parse() on its contents
   - Return (True, "") on success, (False, error_message) on SyntaxError or FileNotFoundError
   - Import: ast, logging, sys, os (stdlib only, matching project pattern)

2. **health_check(server_path: str, timeout: int = 10) -> tuple[bool, str]**
   - Spawn `python3 <server_path>` as a subprocess with stdin=PIPE, stdout=PIPE, stderr=PIPE
   - Send a JSON-RPC initialize request on stdin: `{"jsonrpc": "2.0", "id": 1, "method": "initialize", "params": {"protocolVersion": "2024-11-05", "capabilities": {}, "clientInfo": {"name": "health-check", "version": "1.0"}}}`
   - Follow with a newline
   - Wait up to `timeout` seconds for a response on stdout
   - Parse the response: if it contains `"result"` with a valid JSON-RPC response, return (True, "")
   - On any error (process crash, timeout, invalid response), return (False, error_message)
   - Always terminate the subprocess in a finally block (process.kill() + process.wait())
   - Note: MCP stdio servers read JSON-RPC from stdin and write to stdout. The base_helpers.py.tmpl redirects application stdout to stderr, keeping stdout clean for JSON-RPC.

3. **record_failure(extension_name: str) -> int**
   - Load registry via registry._load_registry()
   - Find the extension entry, increment its "consecutive_failures" counter (default 0)
   - Save via registry._save_registry()
   - Return the new failure count
   - Use REGISTRY_PATH from registry module (import extensions.registry as registry)

4. **clear_failures(extension_name: str)**
   - Load registry, set "consecutive_failures" to 0 for the extension
   - Save registry

5. **check_and_disable(extension_name: str, max_failures: int = 3) -> bool**
   - Call record_failure() to increment and get count
   - If count >= max_failures, set the extension's "enabled" field to False in registry
   - Log a warning when disabling
   - Return True if extension was disabled, False otherwise

Module-level setup:
- logging.basicConfig(stream=sys.stderr, level=logging.INFO)
- logger = logging.getLogger("extension-validator")
- Follow the exact same patterns as generator.py and registry.py (stdlib only, monkeypatchable constants)
