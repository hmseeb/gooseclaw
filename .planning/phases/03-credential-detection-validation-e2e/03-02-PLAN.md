Create docker/extensions/detector.py with these functions:

1. **detect_credentials(text: str) -> list[dict]**
   - Scan text for credential-like patterns using regex
   - Patterns to detect (return list of matches):
     - API keys: strings matching `[A-Za-z0-9_\-]{20,}` that appear after keywords like "key", "token", "api", "secret", "password" (case insensitive)
     - App passwords: 16-character strings of lowercase letters (e.g., Fastmail/Google app passwords like "abcdefghijklmnop" or "abcd efgh ijkl mnop" with spaces)
     - Bearer tokens: strings starting with "Bearer " followed by a long alphanumeric string
     - Known prefixes: strings starting with "sk-", "pk-", "ghp_", "gho_", "xoxb-", "xoxp-", "AKIA" (AWS), "glpat-" (GitLab)
   - Each match returns: `{"value": "<the credential>", "type": "<detected_type>", "confidence": <0.0-1.0>}`
   - detected_type is one of: "api_key", "app_password", "bearer_token", "aws_key", "github_token", "slack_token", "gitlab_token"
   - Strip surrounding whitespace and quotes from detected values
   - Return empty list if no credentials detected

2. **classify_credential(credential: dict, user_hint: str = "") -> dict**
   - Takes a detected credential dict and optional user hint (e.g., "this is for fastmail" or "github token")
   - Returns: `{"template": "<template_name>", "vault_prefix": "<prefix>", "vault_keys": [<key_paths>], "extension_name": "<name>", "description": "<desc>"}`
   - Classification rules:
     - If type == "app_password" and hint contains "mail"/"fastmail"/"email"/"imap"/"smtp": template="email_imap", vault_prefix from hint or "email", vault_keys=["{prefix}.imap_host", "{prefix}.imap_user", "{prefix}.app_password", "{prefix}.smtp_host", "{prefix}.smtp_user", "{prefix}.smtp_password"]
     - If type == "github_token" or hint contains "github": template="rest_api", vault_prefix="github", vault_keys=["github.api_key"], extra_subs={"auth_type": "bearer", "service_description": "GitHub API"}
     - Default (any API key/token): template="rest_api", vault_prefix derived from hint or "custom_api", vault_keys=["{prefix}.api_key"]
   - Return includes an "extra_subs" key with any template-specific substitutions

3. **credential_to_extension(credential_value: str, classification: dict) -> dict**
   - End-to-end pipeline: vault -> generate -> validate -> register
   - Steps:
     a. Vault the credential: subprocess.run(["secret", "set", key_path, credential_value]) for the primary key. For email templates with multiple keys, only vault the app_password key; return a prompt dict asking the user for the remaining fields (host, user, etc.)
     b. Generate: call register_generated_extension() from gateway module (lazy import like gateway.py does)
     c. Validate syntax: call validator.validate_syntax() on the generated server_path. If fails, return {"success": False, "error": "Syntax validation failed: {msg}", "stage": "validation"}
     d. Health check: call validator.health_check() on server_path. If fails, call validator.check_and_disable() and return {"success": False, "error": "Health check failed: {msg}", "stage": "health_check"}. Note: health check may fail in environments without vault access (e.g., during generation before vault is populated). Log warning but do NOT auto-disable on first health check failure during initial setup.
     e. On success: call validator.clear_failures() and return {"success": True, "extension_name": name, "server_path": path}
   - On any exception, return {"success": False, "error": str(e), "stage": "unknown"}

Module-level setup:
- logging.basicConfig(stream=sys.stderr, level=logging.INFO)
- logger = logging.getLogger("extension-detector")
- import re, subprocess, logging, sys, os (stdlib only in module scope)
- Lazy imports for gateway, validator, registry (avoid circular imports, matching gateway.py pattern)

```python
def handle_credential_setup(data):
    """Handle credential setup request from AI.
    
    Called when the AI detects a credential and user has confirmed.
    data: {"credential_value": str, "credential_type": str, "user_hint": str, 
           "vault_prefix": str (optional), "template": str (optional)}
    
    Returns dict with success/error status.
    """
    from extensions.detector import detect_credentials, classify_credential, credential_to_extension
    
    cred_value = data.get("credential_value", "").strip()
    if not cred_value:
        return {"success": False, "error": "No credential value provided"}
    
    # If template/prefix explicitly provided, use those
    # Otherwise, classify from type + hint
    classification = data.get("classification")
    if not classification:
        cred_dict = {"value": cred_value, "type": data.get("credential_type", "api_key"), "confidence": 1.0}
        classification = classify_credential(cred_dict, data.get("user_hint", ""))
    
    # Override vault_prefix if user specified
    if data.get("vault_prefix"):
        classification["vault_prefix"] = data["vault_prefix"]
    if data.get("template"):
        classification["template"] = data["template"]
    
    return credential_to_extension(cred_value, classification)
```
