**3. Add `pollStartupStatus()` JavaScript function:**

```javascript
let _startupPollTimer = null;

function updateStartupStep(stepId, state) {
  // state: 'pending' | 'active' | 'done' | 'error'
  const el = document.getElementById(stepId);
  if (!el) return;
  el.className = 'startup-step' + (state !== 'pending' ? ' ' + state : '');
}

async function pollStartupStatus() {
  const titleEl = document.getElementById('successTitle');
  const errorArea = document.getElementById('startupError');
  const errorDetail = document.getElementById('startupErrorDetail');
  const openChatBtn = document.querySelector('#step-success .btn-primary');

  if (openChatBtn) openChatBtn.style.display = 'none';

  // initial state: config step active
  updateStartupStep('ss-config', 'active');
  updateStartupStep('ss-starting', 'pending');
  updateStartupStep('ss-ready', 'pending');

  let attempts = 0;
  const maxAttempts = 60; // 2 min max

  _startupPollTimer = setInterval(async () => {
    attempts++;
    if (attempts > maxAttempts) {
      clearInterval(_startupPollTimer);
      titleEl.textContent = 'Startup timed out';
      updateStartupStep('ss-config', 'done');
      updateStartupStep('ss-starting', 'error');
      errorArea.style.display = 'block';
      errorDetail.textContent = 'Timed out waiting for goose web to start. Check container logs.';
      return;
    }

    try {
      const resp = await fetch('/api/setup/status');
      const status = await resp.json();

      if (status.state === 'starting') {
        titleEl.textContent = 'Starting agent...';
        updateStartupStep('ss-config', 'done');
        updateStartupStep('ss-starting', 'active');
        updateStartupStep('ss-ready', 'pending');
      } else if (status.state === 'ready') {
        clearInterval(_startupPollTimer);
        titleEl.textContent = 'Agent is ready!';
        updateStartupStep('ss-config', 'done');
        updateStartupStep('ss-starting', 'done');
        updateStartupStep('ss-ready', 'done');
        if (openChatBtn) openChatBtn.style.display = '';
        document.getElementById('successOrb').textContent = '\u2705';
      } else if (status.state === 'error') {
        clearInterval(_startupPollTimer);
        titleEl.textContent = 'Failed to start';
        updateStartupStep('ss-config', 'done');
        updateStartupStep('ss-starting', 'error');
        updateStartupStep('ss-ready', 'pending');
        errorArea.style.display = 'block';
        errorDetail.textContent = status.error || status.message || 'Unknown error. Check container logs.';
      }
      // idle state means save hasn't triggered start yet — keep polling
    } catch(e) {
      // network error — keep polling (server might be restarting)
    }
  }, 2000);
}
```

**1. Add a recovery page/section:**

The recovery UI should be accessible at `/setup?recover` or shown when the user gets a 401 on the setup page. Since gateway.py sends `WWW-Authenticate: Basic realm="gooseclaw setup"` on 401, the browser shows a native auth dialog. After the user cancels the dialog, they see "Authentication required" text.

Better approach: Add a recovery section within setup.html that activates when `?recover` is in the URL. The `/setup?recover` path is already served by gateway.py's `handle_setup_page()` which serves setup.html regardless of query params. BUT: it requires auth if setup.json exists. So we need gateway.py to allow unauthenticated access specifically for `/setup?recover`.

**Wait — gateway.py changes go in Plan 01.** Since Plan 02 only modifies setup.html, we need to handle this differently.

**Revised approach:** The auth recovery UI is a standalone section within setup.html. It's displayed when `window.location.search` contains `recover`. Gateway already serves `/setup` and any subpath — and the recovery API endpoint `/api/auth/recover` does NOT require auth (it's authenticated by the recovery secret itself).

However, the issue is: if setup.json exists, `handle_setup_page()` requires auth before serving setup.html at all. The locked-out user can't even load the page.

**Solution:** Add a small modification to gateway.py `handle_setup_page()` — if the path is `/setup?recover` or the query contains `recover`, skip the auth check and serve setup.html. This is safe because:
- The recovery form only calls `/api/auth/recover` which validates the recovery secret
- No config data is exposed on the recovery screen
- The recovery endpoint is rate-limited

```javascript
async function submitRecovery() {
  const secret = document.getElementById('recoverySecret').value.trim();
  const resultEl = document.getElementById('recoveryResult');
  if (!secret) {
    resultEl.style.display = 'block';
    resultEl.innerHTML = '<div class="alert alert-error">Please enter the recovery secret.</div>';
    return;
  }

  resultEl.style.display = 'block';
  resultEl.innerHTML = '<div style="color:var(--text-secondary)">Verifying...</div>';

  try {
    const resp = await fetch('/api/auth/recover', {
      method: 'POST',
      headers: {'Content-Type': 'application/json'},
      body: JSON.stringify({ secret }),
    });
    const data = await resp.json();
    if (data.success && data.auth_token) {
      resultEl.innerHTML = `
        <div class="alert alert-success" style="text-align:left">
          <strong>Token reset successfully!</strong><br>
          <div class="token-box" style="margin-top:12px;display:block">
            <div class="token-label">New Auth Token &mdash; save this</div>
            <code>${data.auth_token}</code>
          </div>
          <p style="margin-top:12px;font-size:13px;color:var(--text-secondary)">
            Use this token to log in. It won't be shown again.<br>
            <a href="/setup" style="color:var(--text-accent)">Go to setup page</a>
          </p>
        </div>`;
    } else {
      resultEl.innerHTML = `<div class="alert alert-error">${data.error || 'Recovery failed.'}</div>`;
    }
  } catch(e) {
    resultEl.innerHTML = '<div class="alert alert-error">Could not reach the server. Try again.</div>';
  }
}
```
