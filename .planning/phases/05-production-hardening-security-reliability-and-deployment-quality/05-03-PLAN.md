<objective>
Eliminate eval injection vulnerability in entrypoint.sh and hash auth tokens instead of storing plaintext. Fix the auth architecture so gateway owns all authentication and goose web uses an internal-only token.

Purpose: The entrypoint.sh `eval "$(python3 -c "...")"` pattern with setup.json values injected directly is a critical shell injection vector. A malicious setup.json value like `"; rm -rf / #` would execute arbitrary commands. The plaintext auth token in setup.json means anyone with volume access has full API access. These are the two most dangerous issues in the codebase.

Output: Safe entrypoint.sh without eval injection, hashed auth tokens in setup.json, gateway-owns-all-auth architecture for goose web.
</objective>

<task type="auto">
  <name>Task 2: Hash auth tokens and implement gateway-owns-all-auth architecture</name>
  <files>docker/gateway.py</files>
  <action>
1. **Add token hashing functions** near the top of gateway.py (after imports):
   ```python
   import hashlib

   def hash_token(token):
       """Hash an auth token using SHA-256 for storage."""
       return hashlib.sha256(token.encode()).hexdigest()

   def verify_token(provided, stored_hash):
       """Verify a provided token against a stored hash."""
       return hashlib.sha256(provided.encode()).hexdigest() == stored_hash
   ```
   Using SHA-256 (not bcrypt) because: Python stdlib only constraint (no bcrypt library), auth tokens are high-entropy random strings (secrets.token_urlsafe), and timing attacks are not a concern for this use case (not passwords).

2. **Modify `handle_save()`** to hash the token before storing:
   - After generating `config["web_auth_token"] = secrets.token_urlsafe(24)`, store TWO things:
     - `config["web_auth_token_hash"] = hash_token(config["web_auth_token"])` -- persisted to setup.json
     - Keep the plaintext token in a local variable for the one-time response
     - Remove `config["web_auth_token"]` from the dict AFTER hashing but BEFORE save_setup() -- so plaintext never hits disk
   - In the response, still return the plaintext token (one-time display to user)
   - If the user PROVIDED a token (not auto-generated), hash it the same way

3. **Modify `get_auth_token()` to return a (token_or_hash, is_hashed) tuple:**
   - Check for `web_auth_token_hash` field first (new format) -> return (hash, True)
   - Fall back to `web_auth_token` for backward compatibility (old installs) -> return (plaintext, False)
   - Env var `GOOSE_WEB_AUTH_TOKEN` -> return (plaintext, False)

4. **Modify `check_auth()`** to compare correctly:
   - Call `get_auth_token()` to get `(stored, is_hashed)`
   - If `is_hashed`: hash the provided token and compare: `hash_token(provided) == stored`
   - If not `is_hashed` (legacy plaintext or env var): compare directly: `provided == stored`

5. **CRITICAL: Fix start_goose_web() to use an internal-only token instead of the user's auth token.**

   The current code calls `get_auth_token()` and passes the result directly to `goose web --auth-token`. After hashing, this would pass a SHA-256 hash as the auth token to goose web. When the gateway proxies requests, the browser sends the user's plaintext password, which would NOT match the hash that goose web was configured with.

   **Fix with gateway-owns-all-auth architecture:**
   - Add a module-level variable: `_INTERNAL_GOOSE_TOKEN = None`
   - In `start_goose_web()`, instead of calling `get_auth_token()`:
     ```python
     global _INTERNAL_GOOSE_TOKEN
     # Generate a random internal token for goose web communication
     # This token is never exposed to users -- gateway handles all user auth
     _INTERNAL_GOOSE_TOKEN = secrets.token_urlsafe(32)
     cmd = ["goose", "web", "--host", "127.0.0.1", "--port", str(GOOSE_WEB_PORT)]
     cmd += ["--auth-token", _INTERNAL_GOOSE_TOKEN]
     ```
   - In `proxy_to_goose()`, add the internal token to proxied requests:
     ```python
     # Replace any user Authorization header with the internal token
     # Gateway already authenticated the user in do_GET/do_POST
     if _INTERNAL_GOOSE_TOKEN:
         auth_value = base64.b64encode(f"user:{_INTERNAL_GOOSE_TOKEN}".encode()).decode()
         headers_to_proxy["Authorization"] = f"Basic {auth_value}"
     ```
   - This means: gateway authenticates users against the hash, then proxies to goose web using the internal token. Users never know the internal token. Goose web is always auth-protected (no `--no-auth` flag needed).

6. **Migration path:** Existing setup.json files with plaintext `web_auth_token` continue to work. On next save, the token gets hashed.

Do NOT modify the entrypoint.sh token handling -- it reads from env vars, not setup.json.
