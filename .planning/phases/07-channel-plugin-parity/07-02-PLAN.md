<objective>
Add per-user concurrency locks and typing indicator support to ChannelRelay so channel plugins get the same concurrency safety and UX as Telegram.

Purpose: CHAN-02 (per-user locks) prevents concurrent goose requests from same user causing session corruption. CHAN-06 (typing indicators) lets channel plugins show "bot is typing" during relay. Both replicate Telegram's relay pattern (lines 4467-4492).

Output: ChannelRelay acquires per-user lock before relay, sends busy message on contention, and fires typing callback in a loop during relay if provided.

**ChannelRelay.__call__** changes -- wrap the relay section (AFTER command interception, which returns early) with lock and typing:

```python
# Per-user lock (CHAN-02)
user_lock = self._state.get_user_lock(user_key)
lock_timeout = 2 if send_fn else 120  # can't notify user without send_fn
if not user_lock.acquire(timeout=lock_timeout):
    if send_fn:
        send_fn("Still thinking... send /stop to cancel.")
    return ""

try:
    # Typing indicator loop (CHAN-06)
    typing_stop = threading.Event()
    if self._typing_cb:
        def _typing_loop():
            while not typing_stop.is_set():
                try:
                    self._typing_cb(user_id)  # use original user_id, not user_key
                except Exception:
                    pass  # buggy callback must not crash relay
                typing_stop.wait(4)
        threading.Thread(target=_typing_loop, daemon=True).start()

    # ... existing relay + active relay tracking code from 07-01 ...

finally:
    typing_stop.set()  # stop typing loop
    user_lock.release()
```

The structure of __call__ after this plan:
1. Command interception (from 07-01) -- returns early, no lock needed
2. Lock acquire (new -- CHAN-02)
3. Typing loop start (new -- CHAN-06)
4. Active relay tracking + relay to goose (from 07-01)
5. Finally: pop active relay, stop typing, release lock

**_load_channel** change (line ~3037):
- When creating ChannelRelay, pass the typing callback: `relay_fn = ChannelRelay(name, typing_cb=channel.get("typing"))`
- Validate typing is callable if provided: `typing_cb = channel.get("typing"); if typing_cb and not callable(typing_cb): print(f"[channels] warn: {name} typing is not callable, ignoring"); typing_cb = None`
