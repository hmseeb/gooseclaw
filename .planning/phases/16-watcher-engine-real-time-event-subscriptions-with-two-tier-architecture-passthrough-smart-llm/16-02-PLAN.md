_process_smart(watcher, data):
- Build user_text from watcher prompt + json.dumps(data)[:4000]
- Check watcher.get("_session_id") for existing session
- If no stored session: call _create_goose_session(), return error on None, store in watcher["_session_id"]
- Call _relay_to_goose_web(user_text, session_id, channel=watcher.get("channel"))
- If relay returns a session error (e.g. "session not found", "session expired"): clear watcher["_session_id"], create new session, retry relay once
- Return response text or error string

Session reuse rationale (Research Pitfall 3): Each _create_goose_session() creates a new session that never gets cleaned up. Storing session_id in the watcher dict and reusing it prevents session accumulation. Creating a new session only on first fire or when the stored one goes stale.

_fire_watcher(watcher, data):
- Dispatch to _process_smart or _process_passthrough based on watcher["smart"]
- Skip if empty message
- Prefix with [watcher_name]
- Call notify_all(full_message, channel=watcher.get("channel"))
- Update last_fired, fire_count, clear last_error
- Wrap in try/except, set last_error on failure
- Call _save_watchers()

_handle_webhook_incoming(webhook_name, body, headers=None):
- Parse body as JSON, fall back to {"raw": body} on JSONDecodeError
- Find matching watchers: type="webhook", enabled=True, source ends with webhook_name
- For each match: if webhook_secret set, verify HMAC-SHA256 (X-Hub-Signature-256 header), skip on mismatch
- Spawn daemon thread per match calling _fire_watcher(watcher, payload)
- Return count of matched watchers

_verify_webhook_signature(secret, body_bytes, signature_header):
- Compute HMAC-SHA256 of body with secret, compare to signature_header (sha256=... format)
- Return bool

_check_feed_watcher(watcher):
- Fetch URL with urllib.request.urlopen (timeout=30)
- Compute sha256 hash of content
- Update last_check timestamp
- If hash == last_hash, return (no change)
- Update last_hash
- Parse content: try JSON first, then _parse_rss(), then raw text
- Apply regex filter if set (on list items)
- Call _fire_watcher(watcher, data)

_parse_rss(content):
- Use xml.etree.ElementTree to parse RSS 2.0 (item elements) and Atom (entry elements)
- Return list of dicts with title, link, description (truncated to 500 chars)
- Return empty list on ParseError

_parse_feed_content(content, url):
- Try json.loads first
- Try _parse_rss if JSON fails
- Fall back to {"text": content.decode()[:2000]}
