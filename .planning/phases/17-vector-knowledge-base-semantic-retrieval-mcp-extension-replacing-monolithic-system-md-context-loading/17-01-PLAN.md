GREEN phase: Create docker/knowledge/__init__.py (empty). Implement docker/knowledge/chunker.py:
- chunk_file(filepath, source_name) -> list of chunk dicts
- Split by ## headers, then by ### subsections within each section
- Intro text before first ### in a section gets its own chunk
- _make_id(source, *parts) generates hierarchical dot-notation IDs: source stem + slugified section parts

Implement docker/knowledge/indexer.py:
- Reads IDENTITY_DIR env var (default "/data/identity")
- Reads KNOWLEDGE_DB_PATH env var (default "/data/knowledge/chroma")

- Calls get_or_create_collection("runtime") to ensure it exists without wiping
- Chunks system.md, onboarding.md, and all *.schema.md in schemas/
- Adds all chunks to system collection via collection.add()
- Prints count of indexed chunks to stdout

All logging in indexer.py uses print() to stdout (not stderr). This is NOT an MCP server, stdout corruption only matters in server.py.

Run tests. All must pass.
