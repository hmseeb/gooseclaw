GREEN phase: Implement docker/knowledge/server.py:

```python
import os, sys, logging
import chromadb
from mcp.server.fastmcp import FastMCP
```

ALL logging to stderr (Pitfall 2 from research). NEVER use print() or bare stdout writes.

Module-level setup:
- CHROMA_PATH from KNOWLEDGE_DB_PATH env var, default "/data/knowledge/chroma"
- PersistentClient at CHROMA_PATH
- system_col = get_or_create_collection("system")
- runtime_col = get_or_create_collection("runtime")
- mcp = FastMCP("knowledge")

knowledge_search(query: str, type: str = "", limit: int = 5) -> str:
- Clamp limit to max 10
- Query both collections with query_texts=[query], n_results=limit
- If type is non-empty, pass where={"type": type} filter
- Merge results, compute score as round(1 - distance, 3)
- Sort by score descending, take top `limit`
- Format output: "[type] (score: X.XXX) key\ncontent\n  refs: ...\n"
- Return "No matching knowledge found." if empty

knowledge_upsert(key: str, content: str, type: str, refs: str = "") -> str:
- Upsert into runtime_col with id=key, documents=[content]
- Metadata: type, source="runtime", section="", namespace="runtime", refs, key
- Return confirmation message with key

knowledge_get(key: str) -> str:
- Try get by id=[key] from both system_col and runtime_col
- Return formatted content with metadata if found
- Return "No chunk found with key: {key}" if not in either

knowledge_delete(key: str) -> str:
- Check if key exists in system_col first. If yes, return refusal: "Cannot delete system chunks (rebuilt on deploy). Key: {key}"
- Check runtime_col. If key exists, delete and return confirmation
- If key not found anywhere, return "No chunk found with key: {key}"

if __name__ == "__main__": mcp.run(transport="stdio")

Run tests. All must pass.
