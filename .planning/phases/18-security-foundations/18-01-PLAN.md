Fix secret.sh: Convert all 4 commands (get, set, list, delete) to pass shell variables via environment variable prefix on the python3 -c invocation instead of string interpolation.

Variables to convert per command:
- get: $VAULT_FILE -> _VAULT_FILE, $DOTPATH -> _DOTPATH (replace '$VAULT_FILE' with os.environ['_VAULT_FILE'], etc.)
- set: $VAULT_FILE -> _VAULT_FILE, $DOTPATH -> _DOTPATH, $VALUE -> _VALUE (critical: replace '''$VALUE''' with os.environ['_VALUE'])
- list: $VAULT_FILE -> _VAULT_FILE
- delete: $VAULT_FILE -> _VAULT_FILE, $DOTPATH -> _DOTPATH

Pattern for each command:
```bash
# BEFORE:
python3 -c "
import yaml, sys
with open('$VAULT_FILE') as f:
"

# AFTER:
_VAULT_FILE="$VAULT_FILE" _DOTPATH="$DOTPATH" python3 -c "
import yaml, sys, os
with open(os.environ['_VAULT_FILE']) as f:
"
```

Add `import os` to each python3 -c block that doesn't have it. Also fix the error message strings that reference $DOTPATH (replace with os.environ['_DOTPATH'] inside f-string or concatenation).

In gateway.py around line 3801, replace `shell=True` with explicit shell invocation:

```python
# BEFORE:
result = subprocess.run(
    command,
    shell=True,
    capture_output=True,
    text=True,
    timeout=timeout,
    cwd=working_dir,
    env=env,

)
```

This is functionally identical but explicit about which shell runs. Avoids Python's platform-dependent shell selection. Job commands legitimately need shell interpretation (pipes, redirects) so we keep the shell but make it explicit.
