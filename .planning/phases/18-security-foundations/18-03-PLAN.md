Three changes in gateway.py:

**1. Add MAX_BODY_SIZE constant** near the top constants (around line 868-874 where DATA_DIR etc are defined):
```python
MAX_BODY_SIZE = 1_048_576  # 1 MB
```

**2. Add body size guard to _read_body()** at line 9705-9707:
```python
def _read_body(self):
    length = int(self.headers.get("Content-Length", 0))
    if length > MAX_BODY_SIZE:
        self.send_json(413, {"error": "Request body too large", "max_bytes": MAX_BODY_SIZE})
        return None
    return self.rfile.read(length) if length > 0 else b""
```

Then grep for every call site of `_read_body()` (there are ~20, found at lines 8373, 8449, 8467, 8505, 8570, 8615, 8646, 8698, 8734, 8776, 8877, 8984, 9023, 9040, 9082, 9120, 9155, 9393, 9476, 9520). At each call site, add a None check immediately after:
```python
body = self._read_body()
if body is None:
    return  # 413 already sent
```

IMPORTANT from Pitfall 4 in research: Every single call site MUST handle None. Missing even one causes a TypeError when passing None to json.loads(). Grep and verify all are handled.

**3. Add body size guard to proxy path** at line 9601-9602:
```python
# BEFORE:
content_length = int(self.headers.get("Content-Length", 0))
body = self.rfile.read(content_length) if content_length > 0 else None

# AFTER:
content_length = int(self.headers.get("Content-Length", 0))
if content_length > MAX_BODY_SIZE:
    self.send_json(413, {"error": "Request body too large", "max_bytes": MAX_BODY_SIZE})
    return
body = self.rfile.read(content_length) if content_length > 0 else None
```

**4. Add Cross-Origin-Opener-Policy to SECURITY_HEADERS** at line 860-866:
```python
SECURITY_HEADERS = {

    "Permissions-Policy": "camera=(), microphone=(), geolocation=()",
    "Cross-Origin-Opener-Policy": "same-origin",
}
```

Note: Referrer-Policy and Permissions-Policy already exist. Only Cross-Origin-Opener-Policy is missing per HARD-04.
