Create `docker/tests/test_entrypoint.py`. Read entrypoint.sh thoroughly first to understand what it does:

1. **Directory creation** - Creates /data/config, /data/secrets, /data/sessions, /data/channels, etc.
2. **Config generation** - Creates default setup.json, vault.yaml if missing
3. **Env rehydration** - Reads setup.json and re-exports provider env vars
4. **Provider detection** - Checks which provider is configured
5. **Password reset** - If GOOSECLAW_RESET_PASSWORD is set, resets the password

The entrypoint.sh is 700 lines so it has substantial logic. Focus on the testable, isolated parts.

**Testing approach:** Run entrypoint.sh sections via subprocess with a tmpdir as DATA_DIR. The script likely needs to be partially executed (it probably starts the gateway at the end). Two options:
a) Source specific functions if the script has them
b) Run the script but kill it after the bootstrap phase (before it starts gateway)
c) Extract the bootstrap section check logic

Read the script to determine the best approach.

**class TestEntrypointDirectoryCreation:**
- `test_creates_data_directories(tmp_path)` - Set DATA_DIR to tmp_path, run the bootstrap portion, assert config/, secrets/, sessions/, channels/ dirs exist.

**class TestEntrypointConfigGeneration:**
- `test_creates_default_setup_json(tmp_path)` - Run bootstrap with empty DATA_DIR, assert config/setup.json is created with expected defaults.
- `test_creates_default_vault(tmp_path)` - Assert secrets/vault.yaml is created.
- `test_preserves_existing_config(tmp_path)` - Write a setup.json before running, assert it's not overwritten.

**class TestEntrypointEnvRehydration:**
- `test_rehydrates_provider_env_vars(tmp_path)` - Write a setup.json with provider config (e.g., {"provider": "openai", "OPENAI_API_KEY": "sk-test"}), run the bootstrap, check that the env var export is in the output or that the script sets it.

**class TestEntrypointProviderDetection:**
- `test_detects_configured_provider(tmp_path)` - Write setup.json with provider, run bootstrap, assert output indicates provider detected.
- `test_no_provider_shows_setup_needed(tmp_path)` - Empty setup, assert output indicates setup needed.

**class TestEntrypointPasswordReset:**
- `test_password_reset_via_env_var(tmp_path)` - Set GOOSECLAW_RESET_PASSWORD=newpass and DATA_DIR=tmp_path, run entrypoint. Assert setup.json is updated with a new hash. This tests SEC-02's behavioral contract.

**Important:** entrypoint.sh ends by exec'ing the gateway. For testing, either:
- Use `timeout 5` to kill after bootstrap
- Set a special env var if the script supports dry-run
- Source specific functions
- Use `bash -c 'source entrypoint.sh; specific_function'` if functions exist

Read the script to determine which approach works. The goal is to test bootstrap logic without starting the full gateway.

**Important:** The script likely uses `set -euo pipefail`. Subprocess tests should check exit codes carefully.
