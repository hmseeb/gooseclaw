<task type="auto">
  <name>Task 1: Generate hash-pinned lock file and update Dockerfile</name>
  <files>docker/requirements.lock, Dockerfile</files>
  <action>
1. Install pip-tools in local env: `pip install pip-tools`

2. Generate requirements.lock with hashes. Because the Docker image is Linux amd64 and local dev may be macOS, generate platform-agnostic output. Run:
   ```
   pip-compile --generate-hashes --output-file=docker/requirements.lock docker/requirements.txt
   ```
   If hash mismatch issues arise on cross-platform wheels, use `--allow-unsafe` flag or generate inside a Docker container matching the base image (Ubuntu 22.04).

3. Verify the lock file contains:
   - Every direct dependency (PyYAML, websocket-client, chromadb, mcp) pinned to exact version (==)
   - Every transitive dependency pinned with exact version
   - At least one `--hash=sha256:` per package line

4. Update Dockerfile line 51-52. Change from:
   ```dockerfile
   COPY docker/requirements.txt /app/docker/requirements.txt
   RUN pip3 install --no-cache-dir -r /app/docker/requirements.txt
   ```
   To:
   ```dockerfile
   COPY docker/requirements.txt docker/requirements.lock /app/docker/
   RUN pip3 install --no-cache-dir --require-hashes -r /app/docker/requirements.lock
   ```

5. Also update the Dockerfile comment on line 13 to mention requirements.lock.
  </action>
  <verify>
    <automated>grep -c "hash=sha256" docker/requirements.lock && grep "require-hashes" Dockerfile</automated>
    <manual>Lock file has hashes for all deps, Dockerfile references requirements.lock</manual>
  </verify>
  <done>requirements.lock exists with hash pins for all direct and transitive dependencies. Dockerfile installs from lock file with --require-hashes.</done>
</task>
