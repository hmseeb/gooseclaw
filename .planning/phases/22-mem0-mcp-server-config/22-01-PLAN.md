  <name>Task 1: Add mem0ai dependency and create shared config module</name>
  <files>docker/requirements.txt, docker/mem0_config.py</files>
  <action>
1. Add to docker/requirements.txt (after the existing chromadb line):
   ```
   mem0ai==1.0.6
   sentence-transformers>=5.0.0
   ```

2. Create docker/mem0_config.py with the shared config builder. Follow the exact code from 22-RESEARCH.md "mem0 Config Builder" section. The module must contain:

   - CONFIG_DIR and SETUP_FILE path constants (read from env, default /data/config)
   - PROVIDER_MAP dict mapping setup.json provider_type values to mem0 LLM provider names:
     anthropic->anthropic, openai->openai, google->google, groq->groq, openrouter->litellm, ollama->ollama, deepseek->deepseek, together->together, azure-openai->azure_openai, litellm->litellm
   - CHEAP_MODELS dict mapping provider to cheapest viable extraction model:
     anthropic->claude-haiku-4-20250414, openai->gpt-4.1-nano, google->gemini-2.0-flash, groq->llama-3.3-70b-versatile, ollama->llama3.2, deepseek->deepseek-chat, together->meta-llama/Llama-3-8b-chat-hf, litellm->gpt-4.1-nano, openrouter->anthropic/claude-3-haiku-20240307, azure-openai->gpt-4o-mini
   - PROVIDER_ENV_KEYS dict mapping provider to API key env var name (ollama=None)
   - _load_setup() function: reads setup.json from CONFIG_DIR, falls back to .bak, returns None on failure
   - build_mem0_config() function: loads setup, maps provider, builds config dict with:
     - vector_store: provider="chromadb", collection_name="mem0_memories", path from MEM0_CHROMA_PATH env or "/data/knowledge/chroma"
     - embedder: provider="huggingface", model="sentence-transformers/all-MiniLM-L6-v2"
     - llm: provider from PROVIDER_MAP, model from CHEAP_MODELS, temperature=0.1, max_tokens=2000
     - version: "v1.1"
   - For openrouter provider: add api_key from OPENROUTER_API_KEY env to llm_config

   CRITICAL: Use "chromadb" not "chroma" for vector_store provider (Pitfall 3 from research).
  </action>

    <manual>Verify the config module imports cleanly and returns expected defaults</manual>
  </verify>
