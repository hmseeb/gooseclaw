2. Create docker/memory/server.py following the exact code from 22-RESEARCH.md "mem0 MCP Server (6 tools)" section, with the existing knowledge/server.py as the structural pattern.

The server must:
- Set MEM0_TELEMETRY=false in os.environ BEFORE importing mem0 (Pitfall 4)
- Configure logging to stderr (same as knowledge/server.py)
- Import build_mem0_config via sys.path manipulation (same parent-dir pattern as research shows)
- Initialize Memory.from_config(config) at module level
- Read MEM0_USER_ID from env (default "default")
- Create FastMCP("mem0-memory") server instance

6 tools to implement:

a. **memory_add(content: str) -> str**: Calls memory.add(messages=[{"role": "user", "content": content}], user_id=USER_ID). Returns json.dumps(result, default=str). Wraps in try/except, logs errors, returns error string on failure.

b. **memory_search(query: str, limit: int = 5) -> str**: Clamps limit to 1-20. Calls memory.search(query=query, user_id=USER_ID, limit=limit). Formats results as "[score] memory_text (id: memory_id)" lines. Returns "No matching memories found." if empty. Access results via results.get("results") or results if it's a list (handle both formats per research open question #2).

c. **memory_delete(memory_id: str) -> str**: Calls memory.delete(memory_id=memory_id). Returns confirmation or error string.

d. **memory_list(limit: int = 10) -> str**: Clamps limit to 1-50. Calls memory.get_all(user_id=USER_ID). Formats as "- memory_text (id: memory_id)" lines. Returns "No memories stored yet." if empty. Slices to limit after fetching.

e. **memory_history(memory_id: str) -> str**: Calls memory.history(memory_id=memory_id). Formats as "[event] old_memory -> new_memory (created_at)" lines. Returns "No history found for this memory." if empty.

f. **memory_get(memory_id: str) -> str**: Calls memory.get(memory_id=memory_id). Returns formatted memory text with metadata, or "Memory not found." if None/empty.

Each tool must have a descriptive docstring with Args section (FastMCP uses these for tool descriptions). Each tool wraps calls in try/except, logs errors to stderr, returns user-friendly error string.

The main block: `if __name__ == "__main__": mcp.run(transport="stdio")`

IMPORTANT: The server follows the knowledge/server.py pattern exactly. Regular Python functions decorated with @mcp.tool(), testable by calling directly.
