  <action>
Create docker/knowledge/migrate_to_mem0.py following the EXACT pattern from the research (24-RESEARCH.md Pattern 1). The script must:

1. Set MEM0_TELEMETRY=false before any mem0 import
2. Define migrate(chroma_path=None, sentinel_path=None) with defaults from env vars
3. Guard: if sentinel exists, print skip message and return 0
4. Open chromadb PersistentClient, try get_collection("runtime"). If not found, touch sentinel and return 0
5. Call col.get(include=["documents", "metadatas"]) to read all entries
6. If no entries, touch sentinel and return 0
7. Import mem0.Memory and mem0_config.build_mem0_config (add parent dir to sys.path first)
8. For each entry: call memory.add(messages=doc, user_id=user_id, infer=False, metadata={"source": "chromadb_migration", "original_key": doc_id, "original_type": meta.get("type", "")})
9. Count successes, log failures, touch sentinel, return count
10. _touch_sentinel writes timestamp to file
11. if __name__ == "__main__": migrate()

Then add TestMem0Migration class to docker/test_knowledge.py with these test methods:
- test_migration_skips_if_sentinel_exists: create sentinel file, call migrate(), assert mem0 Memory never instantiated
- test_migration_handles_missing_collection: mock chromadb client.get_collection to raise exception, call migrate(), assert sentinel created, assert returns 0
- test_migration_handles_empty_collection: mock collection.get returning empty ids, call migrate(), assert sentinel created
- test_migration_uses_infer_false: mock collection with 2 entries, mock mem0.Memory, call migrate(), assert memory.add called with infer=False for each entry (this covers MIG-02)
- test_migration_creates_sentinel_after_success: mock collection with 1 entry, mock mem0.Memory, call migrate(), assert sentinel file exists with timestamp content
- test_migration_continues_on_single_failure: mock collection with 3 entries, make memory.add raise on 2nd call, assert returns 2 (not 3), sentinel still created

Use chromadb.EphemeralClient() or MagicMock for the chromadb source. Mock mem0.Memory and mem0.Memory.from_config via unittest.mock.patch. Use tmp_path for sentinel file paths. Follow the existing test_knowledge.py patterns (look at TestMigrateMemory class for the mock conventions).
