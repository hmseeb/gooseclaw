```dockerfile
# install Neo4j Community + OpenJDK 21 for graph memory (Phase 25)
RUN wget -O - https://debian.neo4j.com/neotechnology.gpg.key | gpg --dearmor -o /etc/apt/keyrings/neotechnology.gpg && \
    echo 'deb [signed-by=/etc/apt/keyrings/neotechnology.gpg] https://debian.neo4j.com stable latest' > /etc/apt/sources.list.d/neo4j.list && \
    apt-get update && \
    apt-get install -y --no-install-recommends neo4j && \
```

**entrypoint.sh changes:**

Add a new section AFTER the mem0 migration block and BEFORE the gateway start. The block:

1. Check if neo4j binary exists (`command -v neo4j`)
2. Create `/data/neo4j` directory and set ownership to neo4j:neo4j
3. Export Neo4j env vars for constrained memory:
   - `NEO4J_AUTH=none` (localhost only, no external access)
   - `NEO4J_server_memory_heap_initial__size=256m`
   - `NEO4J_server_memory_heap_max__size=512m`
   - `NEO4J_server_memory_pagecache__size=128m`
   - `NEO4J_server_directories_data=/data/neo4j`
   - `NEO4J_server_analytics_enabled=false`
4. Start neo4j in background: `neo4j console &`
5. Wait loop (max 60 seconds) checking `neo4j status` for readiness
6. On success: `export NEO4J_ENABLED=true` and log ready message
7. On failure: log warning, do NOT set NEO4J_ENABLED (graph memory stays disabled), continue startup

The env var naming follows Neo4j convention: `NEO4J_` prefix, dots become single underscores, literal underscores become double underscores.

Critical: Neo4j must start BEFORE the gateway so the Bolt port is ready when mem0 initializes. The `NEO4J_ENABLED` env var is the gate for mem0_config.py to include graph_store.

Use this exact pattern for the neo4j block in entrypoint.sh:

```bash
# ---- neo4j graph database (background) ----
if command -v neo4j &>/dev/null; then
    echo "[neo4j] starting graph database..."
    mkdir -p /data/neo4j
    chown -R neo4j:neo4j /data/neo4j 2>/dev/null || true

    export NEO4J_AUTH=none
    export NEO4J_server_memory_heap_initial__size=256m
    export NEO4J_server_memory_heap_max__size=512m
    export NEO4J_server_memory_pagecache__size=128m
    export NEO4J_server_directories_data=/data/neo4j
    export NEO4J_server_analytics_enabled=false

    neo4j console &
    NEO4J_PID=$!

    echo "[neo4j] waiting for bolt://localhost:7687..."
    NEO4J_READY=false
    for i in $(seq 1 60); do
        if neo4j status 2>/dev/null | grep -q "running"; then
            NEO4J_READY=true
            echo "[neo4j] ready (${i}s)"
            break
        fi
        if ! kill -0 "$NEO4J_PID" 2>/dev/null; then
            echo "[neo4j] FAILED to start (process exited)"
            break
        fi
        sleep 1
    done

    if [ "$NEO4J_READY" = "true" ]; then
        export NEO4J_ENABLED=true
        echo "[neo4j] graph memory enabled"
    else
        echo "[neo4j] graph memory disabled (neo4j not ready)"
    fi
else
    echo "[neo4j] not installed, graph memory disabled"
fi
```

**mem0_config.py changes:**

Add a conditional `graph_store` section at the end of `build_mem0_config()`, AFTER the existing config dict is built and BEFORE `return config`. Only include graph_store when `NEO4J_ENABLED` env var is set to a truthy value:

```python
    # Graph store (Neo4j) - optional, enabled when NEO4J_ENABLED is set by entrypoint
    if os.environ.get("NEO4J_ENABLED", "").lower() in ("true", "1", "yes"):
        config["graph_store"] = {
            "provider": "neo4j",
            "config": {
                "url": os.environ.get("NEO4J_URL", "bolt://localhost:7687"),
                "username": os.environ.get("NEO4J_USERNAME", "neo4j"),
                "password": os.environ.get("NEO4J_PASSWORD", ""),
            }
        }
```

This gates graph memory behind the env var that entrypoint.sh only sets after Neo4j starts successfully. If Neo4j fails, the var is unset, and mem0 runs in vector-only mode.
