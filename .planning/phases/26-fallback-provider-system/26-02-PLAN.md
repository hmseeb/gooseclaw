In the Python rehydration script inside entrypoint.sh (the `python3 -c "..."` block starting at line 235), add after the lead/worker multi-model exports (around line 321, before the closing `"` of the python script):

```python
# fallback provider config -> export as JSON env vars for gateway.py
import json as _json
fb = c.get('fallback_providers', [])
if fb and isinstance(fb, list):
    print(f'export FALLBACK_PROVIDERS={shlex.quote(_json.dumps(fb))}')
m0fb = c.get('mem0_fallback_providers', [])
if m0fb and isinstance(m0fb, list):
    print(f'export MEM0_FALLBACK_PROVIDERS={shlex.quote(_json.dumps(m0fb))}')
```

This exports the fallback arrays as JSON-encoded env vars. The gateway can then read them via `os.environ.get("FALLBACK_PROVIDERS")` and `json.loads()` as an alternative to reading setup.json directly. This matches the existing pattern where setup.json values are rehydrated into env vars for portability.

Note: The gateway will primarily read from setup.json at runtime (via `load_setup()`), but the env vars serve as a backup for scenarios where setup.json isn't available or for debugging.

Also add `import json as _json` at the top of the python script if not already present. Check existing imports first. The script already imports `json` on line 237 (`import json, os, shlex`), so use `json` directly instead of `_json`.
