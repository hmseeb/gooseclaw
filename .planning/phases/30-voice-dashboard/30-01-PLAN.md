**JavaScript (inline in `<script>`):**

1. **State Machine:**
   ```javascript
   const STATE = {
     DISCONNECTED: 'disconnected',
     CONNECTING: 'connecting',
     READY: 'ready',
     LISTENING: 'listening',
     THINKING: 'thinking',
     SPEAKING: 'speaking',
   };
   ```
   - `let currentState = STATE.DISCONNECTED;`
   - `function setState(newState)` updates `currentState`, sets `data-state` on mic button, updates status text, and logs state transitions

2. **WebSocket Manager:**
   - `async function connectVoice()`:
     - Sets state to CONNECTING
     - Fetches token from `/api/voice/token` via fetch. On failure, shows error and sets state DISCONNECTED
     - Opens WebSocket to `${proto}//${location.host}/ws/voice?token=${token}`
     - Sets `ws.binaryType = 'arraybuffer'`
     - `ws.onopen`: log only (wait for ready message)
     - `ws.onmessage`: route text messages to `handleProtocolMessage(JSON.parse(event.data))`, binary to `handleAudioData(new Uint8Array(event.data))` (stub for Plan 02)
     - `ws.onclose`: setState(DISCONNECTED), show error if code !== 1000 with "Connection lost. Tap mic to reconnect."
     - `ws.onerror`: show generic connection error
   - `function disconnectVoice()`:
     - If ws exists and readyState is OPEN or CONNECTING, close with code 1000
     - setState(DISCONNECTED)
     - Clean up resources (stub comment for Plan 02 audio cleanup)
   - `function handleProtocolMessage(msg)`:
     - `switch(msg.type)`:
       - `'ready'`: setState(READY)
       - `'transcript'`: stub `addTranscript(msg.speaker, msg.text)` (Plan 03)
       - `'interrupted'`: stub `handleInterruption()` (Plan 02)
       - default: log unknown message type

3. **Stub functions** (implemented in Plans 02 and 03):
   - `function handleAudioData(data) {}` - Plan 02
   - `function addTranscript(speaker, text) {}` - Plan 03
   - `function handleInterruption() {}` - Plan 02

4. **Error Handler:**
   - `function showError(message, duration = 8000)`:
     - Shows error banner at top with message text
     - Auto-dismisses after duration (clearTimeout on subsequent calls)
     - Close button on banner to dismiss manually
   - `function clearError()`: hides banner

5. **Mic Button Handler:**
   - On click, behavior depends on currentState:
     - DISCONNECTED: call `connectVoice()`
     - CONNECTING: do nothing (already connecting)
     - READY: stub `startCapture()` call (Plan 02 implements). For now, just log "Audio capture not yet implemented"
     - LISTENING: stub `stopCapture()` call (Plan 02 implements). For now, just log
     - THINKING: do nothing
     - SPEAKING: stub barge-in (Plan 02 implements). For now, just log
   - Long-press on mic button: do nothing yet (Plan 02 implements push-to-talk)

6. **Disconnect button** in header: calls `disconnectVoice()`

7. **Status text map:**
   ```javascript
   const STATUS_TEXT = {
     disconnected: 'Tap to connect',
     connecting: 'Connecting...',
     ready: 'Tap to speak',
     listening: 'Listening...',
     thinking: 'Thinking...',
     speaking: 'AI is speaking',
   };
   ```

All JavaScript uses strict mode (`'use strict';` at top of script block). No external imports, no module syntax, no CDN references.
