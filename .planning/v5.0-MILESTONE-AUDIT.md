## Tech Debt (3 items)

### HIGH: Neo4j readiness check may fail in Docker

**File:** `docker/entrypoint.sh` line 613
**Issue:** `neo4j status | grep "running"` relies on systemd service layer. Docker has no systemd. If check always fails, NEO4J_ENABLED never exports and graph memory silently disabled.
**Fix:** Replace with bolt port check: `python3 -c "import socket; socket.create_connection(('localhost',7687),1)"` or `nc -z localhost 7687`
**Affects:** GRAPH-01, GRAPH-02

### MEDIUM: Memory writer off by default

**File:** `docker/gateway.py` line 2342
**Issue:** `memory_writer_enabled` defaults to `False`. Automatic post-session extraction never fires on fresh deploys without opt-in.
**Fix:** Either default to `True` or document opt-in requirement clearly.
**Affects:** GW-01

### LOW: NEO4J_PASSWORD not exported

**File:** `docker/entrypoint.sh` (absent)
**Issue:** `mem0_config.py` defaults NEO4J_PASSWORD to `""`. Untested whether neo4j driver accepts blank password against auth-disabled Neo4j.
**Fix:** Add explicit comment + integration test.
**Affects:** GRAPH-02
