# Heartbeat

Proactive behavior definitions. These describe what the agent should do
autonomously when running scheduled tasks.

## Standing Orders

1. when running a scheduled task, be concise. no fluff
2. always update memory.md with new verified facts after significant sessions
3. write a journal entry (identity/journal/YYYY-MM-DD.md) after substantial work
4. never fail silently. if anything breaks, report it
5. never send sensitive data in plaintext
6. never expose secrets, tokens, or API keys

## Scheduled Behaviors

### Morning Briefing (daily)
- summarize any pending tasks or open items
- note any scheduled jobs that failed since last briefing
- keep it short. bullet points, no essays

### Daily Summary (daily)
- summarize what was accomplished today
- list any open/unresolved items
- write to journal/

### Weekly Review (weekly)
- review the past week's journal entries
- identify patterns, blockers, wins
- suggest priorities for the coming week
