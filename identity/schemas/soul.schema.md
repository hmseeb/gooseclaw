# Soul Schema

AGENT-EVOLVING file. Written during onboarding, then GROWS as the agent learns.

## Rules

- Updates MUST be ADDITIVE. Never full-rewrite the file.
- Add new observations under the right section. Keep it terse.
- Target: 300-800 words populated. Hard max: 1500 words.
- Preserve all section headers even if empty.

## Sections

| Section | Seeded at | Growth | What goes here |
|---------|-----------|--------|----------------|
| Identity | onboarding | rarely changes | name, role, one-line philosophy |
| Personality | onboarding | refined over time | tone (casual/blunt/professional), default casing, humor style, verbosity default, slang/language patterns |
| Communication Patterns | onboarding | GROWS | formats that land well, response length sweet spots, phrasing patterns that get good reactions |
| Decision Framework | onboarding | refined | when to act vs ask, how to handle ambiguity, priority order when goals conflict |
| Strengths | never | GROWS | task types where you consistently deliver, tools you wield well, domains with good context |
| Weaknesses & Pitfalls | never | GROWS | patterns that annoy the user, common failure modes, things you tend to get wrong |
| Learned Behaviors | never | GROWS most | procedural memory. concrete "when X, do Y" rules. extracted from experience, corrections, observed patterns. critical for headless tasks |
